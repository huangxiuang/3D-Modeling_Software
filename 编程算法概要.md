## 1. 整体架构

项目采用 **PyQt5 + PyVista (VTK)** 技术栈，实现 3D 大场景可视化与目标建模。

### 分层设计

```
┌─────────────────────────────────────────────────────┐
│  3DSceneSoftware_test2.py     入口脚本               │
├─────────────────────────────────────────────────────┤
│  src/main_window.py           GUI主窗口 + 逻辑控制   │
│  src/scene_builder.py         3D场景数据工厂         │
│  src/config.py                配置持久化              │
│  src/measurement.py           测距/测角工具           │
│  src/collision.py             AABB碰撞检测            │
│  src/interaction.py           交互模式枚举             │
├─────────────────────────────────────────────────────┤
│  PyVista (VTK)                3D渲染引擎              │
│  PyQt5                        GUI框架                 │
└─────────────────────────────────────────────────────┘
```

### 数据流

```
用户操作 (鼠标/键盘/滑块)
    ↓
main_window.py (模式派发)
    ├── NORMAL 模式 → 3D场景交互 (旋转/平移/缩放)
    ├── MEASURE 模式 → measurement.py (测距/测角)
    └── WAYPOINT 模式 → 路径点采集
    ↓
scene_objects dict (场景数据)
    ↓
_apply_obj_transform_to_actor() (numpy齐次变换矩阵)
    ↓
VTK渲染管道 (pyvista plotter)
```

## 2. src/scene_builder.py — 场景构建

### 2.1 地形生成算法

**方法：高斯函数叠加 + 河道雕刻**

地形高程使用多峰高斯函数（Gaussian Function）叠加生成：

```python
Z = (
    5.0 * np.exp(-((X - 4) ** 2 + (Y - 3) ** 2) / 12)   # 主峰
    + 3.5 * np.exp(-((X + 3) ** 2 + (Y - 4) ** 2) / 9)  # 次峰
    + 2.0 * np.exp(-((X - 1) ** 2 + (Y + 2) ** 2) / 15) # 缓坡
)
```

**数学形式：**
$$f(x,y) = A_1 e^{-\frac{(x-\mu_{x1})^2 + (y-\mu_{y1})^2}{\sigma_1^2}} + A_2 e^{-\frac{(x-\mu_{x2})^2 + (y-\mu_{y2})^2}{\sigma_2^2}} + A_3 e^{-\frac{(x-\mu_{x3})^2 + (y-\mu_{y3})^2}{\sigma_3^2}}$$

- 使用 60×60 网格（`np.linspace(-10, 10, 60)`）
- 3个高斯峰分别模拟主山丘、辅山丘和缓坡
- 标准差（分母 12、9、15）控制山体宽度

**河道雕刻：**
```python
dist = abs(X[i, j] - 3.0 * np.sin(Y[i, j] * 0.4))
if dist < river_width:
    t = dist / river_width
    Z[i, j] = Z[i, j] * (0.05 + 0.15 * t) - river_depth * (1 - t)
```
- 河道路径为正弦曲线：`x = 3.0 * sin(y * 0.4)`
- 中心凹陷深度 0.7 单位，边界平滑过渡
- 效果：沿正弦路径的河谷地貌

**数据存储：**
- 网格类型：`pv.StructuredGrid(X, Y, Z)`
- 高程标量：`grid["elevation"] = Z.flatten(order="F")`（Fortran 顺序展平，匹配 VTK 内部点序）
- 原始 Z 值保存在 `extra["original_z"]` 用于保存/载入

### 2.2 河流生成

**方法：结构化网格贴合正弦路径**

- 沿 Y 轴从 -10 到 10 切 100 个点
- 宽度方向 12 个点（从 -0.8 到 0.8）
- 中心线为 `x = 3.0 * sin(y * 0.4)`
- Z 值恒为 0（平面）

### 2.3 植被生成

**方法：点云沿河岸分布**

- 在河流两侧各生成 100 个点
- 左岸：`x = 3.0 * sin(y * 0.4) - 0.88`
- 右岸：`x = 3.0 * sin(y * 0.4) + 0.88`
- 类型：`pv.PolyData` 点云，以 `add_points` 渲染为绿色点精灵

### 2.4 飞机制造（多部件粘合）

**方法：基本几何体拼合（Constructive Solid Geometry）**

飞机由 **7个子部件** 通过 `merge()` 组合而成：

| 部件 | 几何体 | 位置 | 说明 |
|------|--------|------|------|
| 机身 (fuselage) | `pv.Cylinder` | 中心 (0,0,0)，方向 X 轴 | 半径 0.16，长 1.6 |
| 机头 (nose) | `pv.Cone` | 前端 x=1.05 | 高 0.5，半径 0.16 |
| 左翼 (wing_left) | `pv.Box` | 平移至 (-0.45) Y | 0.6×0.9×0.05 |
| 右翼 (wing_right) | `pv.Box` | 平移至 (+0.45) Y | 0.6×0.9×0.05 |
| 左平尾 (ht_left) | `pv.Box` | 尾部左侧 | 小平尾 |
| 右平尾 (ht_right) | `pv.Box` | 尾部右侧 | 小平尾 |
| 垂尾 (vt) | `pv.Box` | 尾部上方 | 高 0.35 |
| 尾喷管 (nozzle) | `pv.Cylinder` | 尾部末端 | 半径 0.14 |

**关键实现细节：**
```python
airplane = fuselage.merge([nose, wing_left, wing_right,
                           ht_left, ht_right, vt, nozzle])
airplane.scale(1.5, inplace=True)
airplane.translate((-6, -2, 8), inplace=True)
```

- `merge()` 使用 VTK 的 `vtkAppendPolyData`，将多个网格合并为单一 `PolyData`
- 合并后整体缩放 1.5 倍，平移到场景位置
- 两架飞机（aircraft 灰色、aircraft2 红色）共用同一建模流程，仅位置和颜色不同

### 2.5 小鸟建模

**方法：球体 + 锥体组合**

- 身体：`pv.Sphere(radius=0.12)`
- 头部：`pv.Sphere(radius=0.06)`，偏移 (0.15, 0, 0.04)
- 左翼/右翼：`pv.Cone` 斜向伸展
- 尾部：`pv.Cone` 向后伸展
- 使用 `merge()` 合并

### 2.6 树建模

**方法：圆柱 + 圆锥**

- 树干：`pv.Cylinder`（高 0.5，半径 0.06）
- 树冠：`pv.Cone`（高 0.5，底部半径 0.2）
- 树干截面压扁：`trunk.points[:, :2] *= 0.5`（直接用 numpy 数组操作网格点坐标）
- 位置计算：读取地形高斯函数在该点的 Z 值，使树根贴合地形表面
- 注意：当前树只放在 (4.5, 2) 一个位置，需要使用实际的 terrain height map 算法放在地形表面（ID-9）

## 3. src/main_window.py — 主窗口与交互逻辑

### 3.1 GUI 框架

- 继承 `QtWidgets.QMainWindow`
- 布局：中央 `pyvistaqt.QtInteractor`（3D 视口） + 左右底部 `QDockWidget`
- 菜单栏：视角、工具、帮助
- 工具栏：快捷按钮

### 3.2 坐标系统切换

**算法：坐标轴标签映射**

支持 4 套坐标系，仅改变坐标轴标签和显示顺序，底层 VTK 数据保持右手系不变：

| 坐标系 | X 轴 | Y 轴 | Z 轴 |
|--------|------|------|------|
| ENU (东-北-天) | 东 (East) | 北 (North) | 天 (Up) |
| FLU (前-左-上) | 前 (Front) | 左 (Left) | 上 (Up) |
| NED (北-东-地) | 北 (North) | 东 (East) | 地 (Down) |
| NWU (北-西-天) | 北 (North) | 西 (West) | 天 (Up) |

### 3.3 飞机姿态控制（局部旋转）

**核心算法：numpy 显式齐次变换矩阵**

最初使用 VTK 的 PreMultiply/PostMultiply 矩阵级联，但发现 VTK 版本间行为不一致。
改为纯 numpy 构建 4×4 齐次变换矩阵：

```
H = | R·s    offset - R·s·orig_center |
    | 0 0 0          1                |
```

实现步骤：
1. 构建旋转矩阵 R = Rz(yaw) · Ry(pitch) · Rx(roll)（Z-Y-X 欧拉角顺序）
2. 构建缩放矩阵 s（对角线缩放）
3. 计算变换：p' = offset + R · s · (p - orig_center)
4. 通过 `vtk.vtkMatrix4x4.SetElement()` 逐元素写入
5. `vtkTransform.SetMatrix()` 设置到 VTK 渲染管道

**欧拉角范围：**
- 航向角 (Yaw)：0°~360°（正北为 0°，顺时针）
- 俯仰角 (Pitch)：-90°~+90°
- 滚转角 (Roll)：-180°~+180°

### 3.4 鼠标交互 & Qt-VTK 坐标转换

**算法：Y 轴翻转**

Qt 事件系统的坐标系原点在**左上角**，VTK 显示坐标系原点在**左下角**。

```python
def _to_vtk_display(self, x, y):
    height = self.render_window.GetSize()[1]
    return (x, height - y)
```

关键转换点：
- `vtkPropPicker.Pick()` 传入 Y 坐标需翻转
- `vtkWorldPointPicker.Pick()` 传入 Y 坐标需翻转
- 鼠标悬停世界坐标拾取同样需要翻转

### 3.5 场景保存/载入

**数据格式：JSON 配对文件**

```
data/
├── aircraft/
│   └── mission1.json     # 所有 aircraft 的 transform 数据
└── terrain/
    └── mission1.json     # 地形高程 + 配置 + 相机位置
```

**保存流程：**
1. 用户输入数据名称（如 `mission1`）
2. 遍历所有包含 "aircraft" 名字的对象，保存其 transform（offset, yaw, pitch, roll, scale）
3. 保存地形 `extra["original_z"]` + 当前配置 + 相机位置

**载入流程：**
1. 用户选择飞行器 JSON
2. 自动加载同名的 terrain JSON
3. 恢复每个 aircraft 的 transform 并应用到 VTK actor
4. 恢复地形网格 Z 值 + 调用 `_rebuild_actor("terrain")` 强制 VTK 刷新
5. 恢复配置项和相机位置

### 3.6 飞行路径动画

**算法：分段线性插值 + 欧拉角计算**

```
飞行路径 = [P0(start), P1(waypoint1), P2(waypoint2), ..., Pn(waypointN)]
         → [(P0→P1), (P1→P2), ..., (Pn-1→Pn)]  分段
```

每段的姿态计算：
```python
dx = wp[i+1].x - wp[i].x
dy = wp[i+1].y - wp[i].y
dz = wp[i+1].z - wp[i].z
yaw = math.degrees(math.atan2(dx, dy)) % 360        # 偏航角，正北为0°
horiz_dist = math.hypot(dx, dy)
pitch = math.degrees(math.atan2(-dz, horiz_dist))    # 俯仰角，clamp ±90°
roll = 0.0                                            # 滚转角暂设为0
```

**动画驱动：**
- `QTimer` 每 50ms 触发 `_flight_tick()`
- 每个 segment 插值步数 = 总步数 / 分段数
- 位置线性插值：`t = step / steps_per_segment; pos = lerp(from, to, t)`
- 姿态直接在段内保持恒定（简化实现）

## 5. src/measurement.py — 测量工具

### 5.1 测距算法

**原理：欧几里得距离**

```python
dist = np.linalg.norm(p2 - p1)
```

两点间直线距离，结果以 3D 文字浮动显示在连线中点位置。

### 5.2 测角算法

**原理：向量夹角**

```python
v1 = p1 - p2  # 顶点 p2 到 p1 的向量
v2 = p3 - p2  # 顶点 p2 到 p3 的向量
cosang = np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))
angle_deg = np.degrees(np.arccos(np.clip(cosang, -1.0, 1.0)))
```

使用余弦定理计算三点夹角，`np.clip` 防止浮点误差导致 acos 域外值。

### 5.3 撤销与清除

- `undo_last()`：弹出最后测量组的 actor index 列表，按逆序逐个删除（保持 index 有效）
- `clear_all()`：遍历所有 actor 删除并清空列表

## 6. src/collision.py — 碰撞检测

**算法：AABB（轴对齐包围盒）相交测试**

```python
b1 = mesh1.GetBounds()  # (xmin, xmax, ymin, ymax, zmin, zmax)
b2 = mesh2.GetBounds()

return (b1[0] <= b2[1] and b1[1] >= b2[0]   # X轴重叠
    and b1[2] <= b2[3] and b1[3] >= b2[2]   # Y轴重叠
    and b1[4] <= b2[5] and b1[5] >= b2[4])  # Z轴重叠
```

分离轴定理（SAT）在轴对齐情况下的特例：三个轴都重叠 = 碰撞。

批量检测使用 `O(n²)` 双重循环，因场景物体数量少（<20），性能可接受。

## 7. src/interaction.py — 交互模式状态机

**方法：枚举 + 模式路由**

```python
class InteractionMode(Enum):
    NORMAL = auto()             # 常规交互 + 对象选择
    MEASURE_DISTANCE = auto()   # 测距模式
    MEASURE_ANGLE = auto()      # 测角模式
    WAYPOINT = auto()           # 路径点采集模式
```

- 每次只有一个模式活跃
- `main_window._current_mode` 跟踪当前模式
- 鼠标左键点击根据当前模式路由到不同处理函数：
  - `NORMAL` → `_on_obj_select_changed()` 对象选择
  - `MEASURE_DISTANCE` / `MEASURE_ANGLE` → `measurement.MeasurementTool.add_point()`
  - `WAYPOINT` → `_add_waypoint()` 路径点记录
