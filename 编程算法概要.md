## 2. src/scene_builder.py — 场景构建

### 2.1 地形生成算法

**方法：高斯函数叠加 + 河道雕刻**

地形高程使用多峰高斯函数（Gaussian Function）叠加生成：

```python
Z = (
    5.0 * np.exp(-((X - 4) ** 2 + (Y - 3) ** 2) / 12)   # 主峰
    + 3.5 * np.exp(-((X + 3) ** 2 + (Y - 4) ** 2) / 9)  # 次峰
    + 2.0 * np.exp(-((X - 1) ** 2 + (Y + 2) ** 2) / 15) # 缓坡
)
```

**数学形式：**

$$f(x,y) = A_1 e^{-\frac{(x-\mu_{x1})^2 + (y-\mu_{y1})^2}{\sigma_1^2}} + A_2 e^{-\frac{(x-\mu_{x2})^2+(y-\mu_{y2})^2}{\sigma_2^2}} + A_3 e^{-\frac{(x-\mu_{x3})^2+(y-\mu_{y3})^2}{\sigma_3^2}}$$

- 使用 60×60 网格（`np.linspace(-10, 10, 60)`）
- 3 个高斯峰分别模拟主山丘、辅山丘和缓坡
- 标准差（分母 12、9、15）控制山体宽度

**河道雕刻：**

```python
dist = abs(X[i, j] - 3.0 * np.sin(Y[i, j] * 0.4))
if dist < river_width:
    t = dist / river_width
    Z[i, j] = Z[i, j] * (0.05 + 0.15 * t) - river_depth * (1 - t)
```

- 河道路径为正弦曲线：`x = 3.0 * sin(y * 0.4)`
- 中心凹陷深度 0.7 单位，边界平滑过渡
- 效果：沿正弦路径的河谷地貌

**数据存储：**
- 网格类型：`pv.StructuredGrid(X, Y, Z)`
- 高程标量：`grid["elevation"] = Z.flatten(order="F")`（Fortran 顺序展平，匹配 VTK 内部点序）
- 原始 Z 值保存在 `extra["original_z"]` 用于保存/载入

### 2.2 河流生成

**方法：结构化网格贴合正弦路径**

- 沿 Y 轴从 -10 到 10 切 100 个点
- 宽度方向 12 个点（从 -0.8 到 0.8）
- 中心线为 `x = 3.0 * sin(y * 0.4)`
- Z 值恒为 0（平面）

### 2.3 植被生成

**方法：点云沿河岸分布**

- 在河流两侧各生成 100 个点
- 左岸：`x = 3.0 * sin(y * 0.4) - 0.88`
- 右岸：`x = 3.0 * sin(y * 0.4) + 0.88`
- 类型：`pv.PolyData` 点云，以 `add_points` 渲染为绿色点精灵

### 2.4 飞机制造（多部件粘合）

**方法：基本几何体拼合（Constructive Solid Geometry）**

飞机由 **7 个子部件** 通过 `merge()` 组合而成：

| 部件 | 几何体 | 位置 | 说明 |
|------|--------|------|------|
| 机身 (fuselage) | `pv.Cylinder` | 中心 (0,0,0)，方向 X 轴 | 半径 0.16，长 1.6 |
| 机头 (nose) | `pv.Cone` | 前端 x=1.05 | 高 0.5，半径 0.16 |
| 左翼 (wing_left) | `pv.Box` | 平移至 (-0.45) Y | 0.6×0.9×0.05 |
| 右翼 (wing_right) | `pv.Box` | 平移至 (+0.45) Y | 0.6×0.9×0.05 |
| 左平尾 (ht_left) | `pv.Box` | 尾部左侧 | 小平尾 |
| 右平尾 (ht_right) | `pv.Box` | 尾部右侧 | 小平尾 |
| 垂尾 (vt) | `pv.Box` | 尾部上方 | 高 0.35 |
| 尾喷管 (nozzle) | `pv.Cylinder` | 尾部末端 | 半径 0.14 |

**关键实现细节：**
```python
airplane = fuselage.merge([nose, wing_left, wing_right,
                           ht_left, ht_right, vt, nozzle])
airplane.scale(1.5, inplace=True)
airplane.translate((-6, -2, 8), inplace=True)
```

- `merge()` 使用 VTK 的 `vtkAppendPolyData`，将多个网格合并为单一 `PolyData`
- 合并后整体缩放 1.5 倍，平移到场景位置
- 两架飞机（aircraft 灰色、aircraft2 红色）共用同一建模流程，仅位置和颜色不同

### 2.5 小鸟建模

**方法：球体 + 锥体组合**

- 身体：`pv.Sphere(radius=0.12)`
- 头部：`pv.Sphere(radius=0.06)`，偏移 (0.15, 0, 0.04)
- 左翼/右翼：`pv.Cone` 斜向伸展
- 尾部：`pv.Cone` 向后伸展
- 使用 `merge()` 合并

### 2.6 树建模（ID-9）

**方法：圆柱 + 圆锥，地形贴合**

- 树干：`pv.Cylinder`（高 0.5，半径 0.06）
- 树冠：`pv.Cone`（高 0.5，底部半径 0.2）
- 树干截面压扁：`trunk.points[:, :2] *= 0.5`（直接用 numpy 数组操作网格点坐标）

**地形贴合算法：**
树根位置通过计算地形在 (4.5, 2.0) 处的高斯函数值确定：

```python
def terrain_z_at(x, y):
    return (
        5.0 * np.exp(-((x - 4)**2 + (y - 3)**2) / 12)
        + 3.5 * np.exp(-((x + 3)**2 + (y - 4)**2) / 9)
        + 2.0 * np.exp(-((x - 1)**2 + (y + 2)**2) / 15)
    )
```

树干底部 `local_z = -0.5` 偏移 +0.5 使树根贴合地形表面，树冠在 terrain 之上自然生长，不会被覆盖。

### 3.2 坐标系统切换

**算法：坐标轴标签映射**

支持 4 套坐标系，仅改变坐标轴标签和显示顺序，底层 VTK 数据保持右手系不变：

| 坐标系 | X 轴 | Y 轴 | Z 轴 |
|--------|------|------|------|
| ENU (东-北-天) | 东 (East) | 北 (North) | 天 (Up) |
| FLU (前-左-上) | 前 (Front) | 左 (Left) | 上 (Up) |
| NED (北-东-地) | 北 (North) | 东 (East) | 地 (Down) |
| NWU (北-西-天) | 北 (North) | 西 (West) | 天 (Up) |

### 3.3 飞机姿态控制（局部旋转）

**核心算法：numpy 显式齐次变换矩阵**

最初使用 VTK 的 PreMultiply/PostMultiply 矩阵级联，但发现 VTK 版本间行为不一致。
改为纯 numpy 构建 4×4 齐次变换矩阵：

```
H = | R·s    offset - R·s·orig_center |
    | 0 0 0          1                |
```

**欧拉角顺序：Z-Y-X**（先偏航、再俯仰、最后滚转）

实现步骤：
1. 构建旋转矩阵 R = Rz(yaw) · Ry(pitch) · Rx(roll)
2. 构建缩放矩阵 s（对角线缩放）
3. 计算变换：p' = offset + R · s · (p - orig_center)
4. 通过 `vtk.vtkMatrix4x4.SetElement()` 逐元素写入
5. `vtkTransform.SetMatrix()` 设置到 VTK 渲染管道

**欧拉角范围：**
- 航向角 (Yaw)：0°~360°（正北为 0°，顺时针）
- 俯仰角 (Pitch)：-90°~+90°（+90° 垂直爬升，-90° 垂直俯冲）
- 滚转角 (Roll)：-180°~+180°

仅 aircraft 类型对象显示姿态控制滑块（非 aircraft 对象隐藏）。

### 3.4 鼠标交互 & Qt-VTK 坐标转换

**算法：Y 轴翻转**

Qt 事件系统的坐标系原点在**左上角**，VTK 显示坐标系原点在**左下角**。

```python
def _to_vtk_display(self, x, y):
    height = self.render_window.GetSize()[1]
    return (x, height - y)
```

关键转换点：
- `vtkCellPicker.Pick()` 主选（逐 cell 精确交叉），`SetTolerance=0.001`
- `vtkPropPicker.Pick()` 备选回退
- `vtkWorldPointPicker.Pick()` 最终保底（焦平面深度，仅 XY 近似）
- 鼠标悬停世界坐标拾取同样需要翻转

### 3.6 飞行路径动画（ID-11/ID-14）

**算法：Catmull-Rom 样条插值 + 相邻段姿态插值**

#### 3.6.1 路径构建

```python
飞行路径 P = [P₀(start), P₁(waypoint1), P₂(waypoint2), ..., Pₙ(waypointN)]
```

每段的方向角和俯仰角由几何关系计算：

```python
dx = P[i+1].x - P[i].x
dy = P[i+1].y - P[i].y
dz = P[i+1].z - P[i].z
yaw = degrees(atan2(dx, dy)) % 360       # 偏航角，正北为 0°
horiz_dist = hypot(dx, dy)
pitch = degrees(atan2(-dz, horiz_dist))   # 俯仰角，clamp ±90°
```

#### 3.6.2 位置插值：Catmull-Rom 样条（ID-14）

替代原始线性插值，产生 C1 连续的光滑曲线：

对于段 [Pᵢ, Pᵢ₊₁]，Catmull-Rom 公式需要 4 个控制点：

```
P₀ = Pᵢ₋₁        (前一个点，首段镜像: 2·Pᵢ − Pᵢ₊₁)
P₁ = Pᵢ           当前段起点
P₂ = Pᵢ₊₁         当前段终点
P₃ = Pᵢ₊₂        (下一个点，末段镜像: 2·Pᵢ₊₁ − Pᵢ)
```

Catmull-Rom 基函数：

$$q(t) = \frac{1}{2} \left[ (2P_1) + (-P_0 + P_2)t + (2P_0 - 5P_1 + 4P_2 - P_3)t^2 + (-P_0 + 3P_1 - 3P_2 + P_3)t^3 \right]$$

其中 t ∈ [0, 1] 为段内插值参数。

#### 3.6.3 姿态插值：相邻段间角度过渡（ID-14）

原始的 yaw/pitch 在段内保持常数，导致段切换时姿态突变。改进：

- **Yaw**：取当前段 yaw 和下一段 yaw，在段内用 `_lerp_angle()` 线性插值（处理 0/360 绕回）
- **Pitch**：取当前段 pitch 和下一段 pitch，在段内线性插值（数值小，无需绕回处理）
- **Roll**：当前版本固定为 0

```python
entry_yaw = seg[i]["yaw"]
exit_yaw  = seg[i+1]["yaw"] if exists else entry_yaw
# 段内 t ∈ [0,1]:
yaw_t = lerp_angle(entry_yaw, exit_yaw, t)
```

角度插值绕回处理：
```python
d = (b - a) % 360
if d > 180: d -= 360
return (a + d * t) % 360
```

**为什么不用四元数 SLERP？**

ID-14 的解决方案中提到了四元数 SLERP 作为姿态插值方案之一，但最终实现选择了欧拉角 lerp，原因如下：

| 考量 | lerp (当前方案) | 四元数 SLERP |
|------|----------------|-------------|
| **角度差 < 90° 时的视觉差异** | 几乎不可察觉 | 几乎不可察觉 |
| **角度差 > 90° 时的中间速度** | 非匀速（中间快两端慢） | 匀速（球面最短路径） |
| **飞机典型段间角度差** | ≤ 90°（路径点航向不会大角度跳变） | — |
| **Roll 维度** | 固定为 0，无需参与插值 | 需从欧拉角→四元数→SLERP→欧拉角，多一层转换 |
| **代码复杂度** | ~10 行，一目了然 | ~50 行（含欧拉角↔四元数转换 + 四元数乘法） |
| **调试难度** | yaw/pitch 数值直观 | w/x/y/z 无法直观判断飞机朝向 |
| **欧拉角→四元数转换开销** | 无 | 每帧 2 次 `from_euler`/`to_euler` |
| **万向锁风险** | pitch ∈ [-90°, 90°] 避开了临界点 | 无此问题 |

**结论**：在当前场景（段间航向差 ≤ 90°、roll=0、pitch 范围避开万向锁）下，欧拉角 lerp 与四元数 SLERP 的实际效果高度接近，而实现复杂度和调试成本远低于后者。若后续需要 banking/roll 耦合或大角度特技机动，可升级为四元数方案。

#### 3.6.4 动画驱动

- `QTimer` 每 `FLIGHT_INTERVAL_MS` = 50ms 触发 `_flight_tick()`
- 从当前段几何构建 `pos = catmull_rom(path, seg_idx, t)` + 姿态插值 → 更新 transform → 同步 UI 滑块 → `_apply_obj_transform_to_actor()`

### 3.7 飞机速度计算数学模型（ID-15）

#### 3.7.1 问题背景

原始实现中所有段使用固定步数（50 步/2.5 秒），导致：
- 短段（10 单位）速度慢，长段（100 单位）速度快，平飞速度不一致
- 没有考虑姿态对速度的影响（爬升应减速、转弯应减速）

#### 3.7.2 速度模型公式

$$V_{\text{有效}} = V_0 \times k_{\text{pitch}} \times k_{\text{turn}}$$

其中：
- **V₀ = 5.0 u/s** —— 平飞巡航速度（常数基准）
- **k_pitch** —— 俯仰因子，反映爬升/俯冲对速度的影响
- **k_turn** —— 转弯因子，反映转弯半径对速度的影响

#### 3.7.3 俯仰因子 `k_pitch`

$$k_{\text{pitch}} = \max(0.30,\; 1.0 - 0.35 \cdot \sin(\theta_{\text{pitch}}))$$

| pitch | sin(pitch) | k_pitch | 效果 |
|-------|-----------|---------|------|
| 0° (平飞) | 0.00 | 1.00 | 基准速度 |
| +30° (爬升) | 0.50 | 0.83 | 减速 17% |
| +60° (陡升) | 0.87 | 0.70 | 减速 30% |
| −30° (俯冲) | −0.50 | 1.17 | 加速 17% |
| −60° (陡降) | −0.87 | 1.30 | 加速 30% |

**物理含义**：推力在垂直方向的分量抵消重力做功，爬升时可用水平推力减少，俯冲时重力辅助加速。

#### 3.7.4 转弯因子 `k_turn`

$$k_{\text{turn}} = \max(0.30,\; 1.0 - 0.006 \times |\dot{\psi}|)$$

其中 $\dot{\psi}$ 为偏航角速率（°/s），近似计算为：

$$\dot{\psi} \approx \frac{|\psi_{i+1} - \psi_i|}{t_{\text{seg}}} \approx \frac{|\Delta\psi| \cdot V_{\text{approx}}}{d}$$

由于 $V_{\text{approx}}$ 依赖 $k_{\text{turn}}$（循环依赖），使用两遍计算：

| yaw_rate | k_turn | 效果 |
|----------|--------|------|
| 0°/s | 1.00 | 直线飞行 |
| 50°/s | 0.70 | 中等转弯，减速 30% |
| 100°/s | 0.30 (clamp) | 急转弯，减速 70% |

#### 3.7.5 两遍计算算法

```python
# 第一遍：估计 pitch 因子，产生近似速度
k_pitch_est = max(0.30, 1.0 - 0.35 * sin(pitch_rad))
V_approx = V₀ * k_pitch_est
t_approx = d / V_approx
yaw_rate = |Δψ| / t_approx    # 估算 yaw 变化率

# 第二遍：使用完整速度模型确定最终步数
V = flight_speed(pitch, yaw_rate)   # = V₀ * k_pitch * k_turn
t_seg = d / V
steps = clamp(t_seg / Δt, MIN_STEPS, MAX_STEPS)
# Δt = 50ms (timer 间隔)
```

**结果**：平飞时无论段距离长短，速度一致；爬升越陡速度越慢；急转弯大幅减速。

## 5. src/measurement.py — 测量工具

### 5.1 测距算法

**原理：欧几里得距离**

```python
dist = np.linalg.norm(p2 - p1)
```

两点间直线距离，结果以 3D 文字浮动显示在连线中点位置。

### 5.2 测角算法

**原理：向量夹角（余弦定理）**

```python
v1 = p1 - p2   # 顶点 p2 到 p1
v2 = p3 - p2   # 顶点 p2 到 p3
cosang = np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))
angle_deg = np.degrees(np.arccos(np.clip(cosang, -1.0, 1.0)))
```

使用余弦定理计算三点夹角，`np.clip` 防止浮点误差导致 acos 域外值。

### 5.3 撤销与清除

- `undo_last()`：弹出最后测量组的 actor index 列表，按逆序逐个删除（保持 index 有效）
- `clear_all()`：遍历所有 actor 删除并清空列表

## 6. src/collision.py — 碰撞检测

**算法：AABB（轴对齐包围盒）相交测试 — 分离轴定理特例**

```python
b1 = mesh1.GetBounds()  # (xmin, xmax, ymin, ymax, zmin, zmax)
b2 = mesh2.GetBounds()

return (b1[0] <= b2[1] and b1[1] >= b2[0]   # X轴重叠
    and b1[2] <= b2[3] and b1[3] >= b2[2]   # Y轴重叠
    and b1[4] <= b2[5] and b1[5] >= b2[4])  # Z轴重叠
```

分离轴定理（SAT）在轴对齐情况下的特例：三个轴都重叠 = 碰撞。

批量检测使用 `O(n²)` 双重循环，因场景物体数量少（<20），性能可接受。
